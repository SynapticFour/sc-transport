# Measurement Record: 2026-03-23-initial

## Environment

- Host OS: macOS (Darwin)
- Kernel/Version: Darwin 25.3.0 (arm64)
- CPU: Apple M4
- Network setup: local loopback only (no external netem in this run)
- Rust version: rustc 1.93.1

## Scenario

- Transport: SSE baseline, QUIC-stream (feature-enabled with fallback behavior), QUIC-datagram experimental
- Event count: 1000 events for throughput and datagram delivery tests
- Subscriber count: 1
- Injected latency/loss: no OS-level netem; application-level simulated loss used for datagram fallback tests

## Results

- Throughput (criterion quick run):
  - `throughput_sse_1sub_1000events`: ~617-630 us
  - `throughput_quic_stream_1sub_1000events`: ~630-651 us
  - `throughput_datagram_1sub_1000events`: ~3.85-3.91 ms
- Latency:
  - `latency_sse_send_to_receive`: ~10.1-10.5 us
- Delivery ratio:
  - `datagram_delivery_no_loss` integration test passed with ratio assertion `> 0.95`
  - `datagram_delivery_5pct_loss` and `datagram_delivery_20pct_loss` simulated tests passed expected ratio bounds
- Fallback triggered:
  - Datagram fallback trigger test passed (`DeliveryStatus::FellBack`)
  - Rolling-loss fallback unit test passed (`simulated_loss_triggers_fallback`)

## Notes

- Observations:
  - In this implementation state, datagram path is currently slower than SSE/QUIC-stream in local quick benchmarks.
  - QUIC loopback roundtrip is exercised in tests but may be environment-sensitive under some sandbox/CI UDP conditions.
- Limitations:
  - These are initial local measurements only, with single-subscriber scenarios and no Linux `tc netem`.
  - Results should not be interpreted as production performance claims.
